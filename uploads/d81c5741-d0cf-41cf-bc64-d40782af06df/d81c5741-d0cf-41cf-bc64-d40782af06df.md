# Enterprise Document Parser — Test Sample

*Comprehensive test fixture for the Anti-Gravity DOCX pipeline*

## 1. Introduction and Overview

This document is designed to test every structural element that the Anti-Gravity pipeline is expected to handle faithfully. The document deliberately includes complex layouts, merged-cell tables, multiple embedded chart images, and a plain photograph to ensure that the chart-heuristic works correctly.

The pipeline converts this document through the following stages:
Docling structural parse → Table extraction → Image OCR (Textract) → Chart semantics (Bedrock Nova Lite) → Markdown + JSON output.

## 2. Text Formatting &amp; Inline Styles

**Bold text Bold text** is used for emphasis. *Italic text* is used for definitions. Underlined text marks important terms.

Reading order across multiple paragraphs is tested here. This paragraph comes immediately after the formatting demo. The pipeline should preserve the logical sequence of all text blocks when converting to Markdown.

A second body paragraph follows. Docling preserves reading order even when the source document has multi-column or text-box layouts by analysing the structural XML rather than visual flow.
